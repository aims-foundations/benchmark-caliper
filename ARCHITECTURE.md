# Pipeline Architecture: Cost Optimization

## Overview

The validity analysis pipeline has been restructured from an all-Opus design to a
**tiered model routing** architecture. The guiding principle: use the cheapest
tool that produces equivalent output for each step.

## Design Principles

### 1. Context Optimization (Primary)
- **Per-page extraction**: Instead of feeding 8-12 page chunks to a single model,
  each PDF page is processed independently by a Haiku subagent. This reduces
  per-call context from ~40K tokens to ~4K tokens.
- **Minimal subagent context**: Each subagent receives only the information it needs.
  The quote spot-check subagent gets 3 quotes + 3 PDF pages, not the entire pipeline state.
- **Script-based composition**: The evaluation prompt is assembled deterministically
  from YAML files, not generated by an LLM reasoning about what to include.

### 2. Model Routing (Secondary)
- **Haiku**: Per-page quote extraction (pattern matching, not reasoning)
- **Sonnet**: Consolidation, YAML structuring, verification, elicitation, persona simulation, web search (moderate reasoning)
- **Opus**: Validity scoring only (deep cross-referencing, cultural reasoning, judgment)
- **Scripts**: Counting, template filling, formatting (zero LLM needed)

### 3. Script Displacement
Three pipeline steps that previously required an LLM now use deterministic scripts:

| Step | Script | What it replaces |
|------|--------|-----------------|
| Quote verification (mechanical) | `scripts/verify_quotes.py` | Opus counting quotes and checking IDs |
| Prompt composition | `scripts/compose_prompt.py` | Opus assembling the evaluation prompt |
| Report formatting | `scripts/format_results.py` | Opus reading JSON and formatting a table |

## File Structure

```
├── scripts/
│   ├── split_pdf.sh          # PDF page splitting (bash)
│   ├── verify_quotes.py      # Mechanical quote verification (python)
│   ├── compose_prompt.py     # Deterministic prompt assembly (python)
│   └── format_results.py     # Result formatting (python)
├── .claude/
│   ├── prompts/
│   │   ├── run_validity_pipeline.md   # Orchestration prompt (routed)
│   │   ├── elicitation_guide.md       # Elicitation logic (Sonnet, Step 1d)
│   │   ├── web_search_guide.md        # Web search strategy (Sonnet, Step 3)
│   │   └── personas.md                # Test personas for automated testing
│   └── skills/
│       └── validity-pdf-extract/
│           └── SKILL.md               # PDF extraction (Haiku+Sonnet)
├── framework.yaml            # Validity dimensions & rubric (user-provided)
├── prompt_template.md        # Evaluation prompt template (user-provided)
├── README.md                 # Project overview (user-provided)
├── benchmarks/               # Benchmark YAMLs (pipeline output)
├── regions/                  # Region YAMLs (pipeline output)
├── papers/                   # PDFs and extracted summaries
└── results/                  # Validity analysis JSON results
```

## Cost Estimation

Approximate token usage per pipeline run (29-page PDF):

| Step | Model | Input tokens | Output tokens |
|------|-------|-------------|---------------|
| Per-page extraction | Haiku × 29 | ~4K × 29 = 116K | ~500 × 29 = 14.5K |
| Consolidation | Sonnet × 1 | ~20K | ~15K |
| YAML structuring | Sonnet × 1 | ~25K | ~15K |
| Text spot-check | Sonnet × 1 | ~5K | ~1K |
| Elicitation (questions) | Sonnet × 1 | ~15K | ~2K |
| Elicitation (summary) | Sonnet × 1 | ~8K | ~3K |
| Persona simulation* | Sonnet × 1 | ~5K | ~1K |
| Region creation | Sonnet × 1 | ~10K | ~5K |
| Web enrichment | Sonnet × 1 | ~15K | ~8K |
| **Scoring** | **Opus × 1** | **~45K** | **~15K** |
| Scripts | — | 0 | 0 |

*Persona simulation only in `--persona` test mode.

**Previous total**: ~6-8 Opus calls, each with 20-50K input tokens.
**New total**: 1 Opus call (scoring) + 7-8 Sonnet calls + 29 Haiku calls + 3 scripts.

## Prerequisites

- Python 3 with PyYAML: `pip install pyyaml`
- pdftk or qpdf (optional, for PDF splitting; falls back to Read tool)
- User must provide: `framework.yaml`, `prompt_template.md`, `README.md`
